# Thinkphp

thinkphp采取MVC架构

下面是一个Controllor的示例

<?php
namespace Home\Controller;
use Think\Controller;
class IndexController extends Controller {
    public function hello(){
        echo 'hello,thinkphp!';
    }
}

在一般的项目结构中，我们会把它写在

application/home/controller/index.php下

Home\IndexController类就代表了Home模块下的Index控制器，而hello操作就是Home\IndexController类的hello（公共）方法。

当访问 http://serverName/index.php/Home/Index/hello 后会输出：

hello,thinkphp!
1

对于Controller我们可以采取如下方式调用

  1. http://serverName/index.php/模块/控制器/操作/[参数名/参数值...]

  2. http://<serverName>/appName/?m=module&a=action&id=1

  3. http://tp5.com/index.php?s=/index/Index/index

# ThinkPHP 3.X

3.2.3 的默认目录结构如下

├─index.php       入口文件

# Thinkphp 6.0

相对于5.1来说，6.0版本目录结构的主要变化是核心框架纳入vendor目录，然后原来的application目录变成app目录。 6.0支持多应用模式部署，所以实际的目录结构取决于你采用的是单应用还是多应用模式，多应用模式部署后，记得删除app目录下的controller目录（系统根据该目录作为判断是否单应用的依据）。
